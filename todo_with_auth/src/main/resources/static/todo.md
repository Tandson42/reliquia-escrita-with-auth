Esta atividade corresponde a parte do Trabalho Final a ser apresentado pelos acadêmicos. Atividade em grupos (manter os mesmos grupos da Atividade 5).

Consiste em elaborar um site que poderá ser local ou publicado em alguma plataforma de hospedagem em nuvem, pública de preferência.

Este site deverá conter, no mínimo, 4(quatro) páginas construídas com HTML, CSS, um Framework CSS. O Framework CSS usado deve ser um dos apresentados pelo grupo na Atividade 5.

O assunto do site pode ser uma página profissional do(s) aluno(s) ou uma empresa/instituição fictícia. Os dados não podem ser copiados de empresas reais.

A navegação entre as páginas do site deve ser por meio de um menu. Links no meio do texto das paginas podem ser usados, mas o menu é obrigatório. Caso o framework CSS faça isso, pode ser usado. 

Uma das páginas deve ser para configuração de elementos visuais do site. Pelo menos 2 elementos. Um deles, obrigatório,  é a paleta de cores. Colocar 12 ou mais opções de paletas ou 6 opções e uma paleta customizada. O outro elemento fica à escolha do grupo. Pode ser layout ou tamanho das fontes, etc. As alterações feitas nesta página devem refletir no site imediatamente. Ou seja, deve-se usar Javascript para isso. Caso o framework CSS faça isso, pode ser usado. 

As alterações na página de configuração devem persistir no lado do cliente. Ou seja, pode-se fechar o navegador e reabrir, e as alterações deve estar com a última escolha feita pelo usuário. Pode-se usar cookies para fazer isso. Outra solução pode ser usada. Caso o framework CSS faça isso, pode ser usado. Se usar cookies, seria bom haver uma mensagem temporária no página inicial indicando que o site usa cookies, para atender a LGPD, assim como fazer os sites profissionais.

O site deve ser responsivo à largura do viewport. Deve-se usar a largura default com um design para navegador de desktop/notebook. Ajustar o design para uma tela com largura menor que 1200px (tablet) usando @media, alterando a orientação dos elementos de horizontal para vertical. Ajustar o design para uma tela com largura menor que 600px (smartphone na vertical) usando @media, alterando caraterísticas como bordas, padding e/ou tamanho das fontes.

A quarta página deve ser uma lista de tarefas, tipo TODO LIST. Deve-se usar o back-end fornecido em sala pelo professor.  Para consumir este webservice deve usar AJAX e utilizar algum framework Javascript/Typescript dentre os vistos em sala, como JQuery ou Angular. O quesito a ser avaliado é o design do front-end. Se o framework CSS escolhido fornecer suporte à conexão com o back-end, pode ser usado.
